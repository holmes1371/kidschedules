# 32. "Completed" checkbox on event cards — 4828713 (design note + ROADMAP plan-block) / 732a0de (server render + classify + --completed CLI + 11 tests) / 3cd394e (Apps Script: complete/uncomplete actions + Completed Events tab) / 863b2f8 (scripts/sync_completed_events.py + 26 unit tests) / 2c373fc (main.py wiring + workflow YAML sync step) / caa6566 (client JS handler + hydration + 5 JS tests) / 1325465 (initial close-out, SHAs recorded) / 636abe0 (CI fix: 7 failures from always-render chip + storage-key alignment) / 3667823 (UX: checkbox at left edge of action row)

Filed 2026-04-27 from Tom. Per-card "completed" affordance so Ellen can mark an event done as it happens, without waiting for the date to pass it off the page. Mirrors the ignore-flow architecture surface-for-surface: same 12-hex event-id, same Apps Script POST/GET round-trip, same ephemeral sheet-derived cache, same optimistic toggle + rollback. Full design at `design/completed-events.md`.

**Resolved decisions (2026-04-27).** Durable persistence (sticks across page reloads and cron rebuilds); cross-device sync via Apps Script (so a phone check propagates to tablet on next refresh); completed supersedes ignore (both Ignore-event AND Ignore-sender suppressed via CSS on `.event-card.completed` — the checkbox is the sole affordance on a completed card); ignored cards do NOT show the completed checkbox (CSS-suppressed via the existing `[data-ignored="1"]` selector pattern, so the two affordances are mutually exclusive on a given card); retirement unchanged (completed cards retire on the normal date-passed threshold alongside uncompleted ones — no early sweep, no second retirement codepath). Sheet-as-single-source-of-truth invariant preserved end-to-end: `completed_events.json` is per-run ephemeral, written by the new sync step into the runner's working dir each cron tick, consumed by `process_events.py`, never committed.

**Architecture.** New "Completed Events" sheet tab with rows `[timestamp, id, name, date]` (no sender column — completion has no per-sender sweep). Apps Script `doPost` extended with `complete` / `uncomplete` action routes, both 12-hex id-validated. `doGet` extended with `?kind=completed`. Sync helper at `scripts/sync_completed_events.py` mirrors `sync_ignored_senders.py` one-for-one apart from validation key. Server-side render: chip rendered conditionally next to the event name when `is_completed=True`; checkbox always rendered (so unchecking restores it after a round-trip), CSS-suppressed on ignored cards. Client JS: `setCompleted` helper toggles class, creates/destroys the chip span, and flips the checkbox state in lockstep; separate `change` listener on `.complete-checkbox` so the new checked state is read reliably without racing the click delegator.

**Mid-feature pivot.** The chip was originally rendered always-in-DOM with CSS gating (mirroring the design's Q7 "always render, CSS-gate" pattern used for the checkbox-on-ignored suppression). That broke 4 unrelated existing tests (3 NEW-badge substring pins on `Name<span class="new-badge">NEW</span></div>` plus 1 location-linkify slice that lost byte margin). Reverted to conditional render with JS chip injection on toggle (commit `636abe0`); cleaner outcome — single CSS rule, no test ripple, JS adds ~10 lines for create/destroy.

**Tests.** Net +37 server-side: 8 classify + render asserts (chip presence/absence, checkbox checked/unchecked, CSS suppression rules in both directions, undated-card parity), 6 JS-substring asserts (storage key, complete/uncomplete actions, rollback toasts, hydration loop, setCompleted helper), 6 parametrized loader tolerance pairs, 13 sync-helper unit tests (normalize, write_if_changed, _fetch happy/error paths, CLI argv plumbing). All 745 tests pass on Linux/CI; render tests fail on Windows + Python 3.14 with the unrelated `%-d` strftime issue (per-session protocol now: run full pytest on a strftime-patched copy before any push — memory note saved).

**Live verification (2026-04-27).** Tom verified end-to-end on his phone: clicking the checkbox flips card to green tint with chip, Ignore buttons disappear, sheet row appears in "Completed Events" tab, unchecking reverses every step. UX tweak landed same session at Tom's request: checkbox positioned at the left edge of the action row (`margin-right: auto` on `.complete-checkbox-wrap`) instead of stacked with ICS / Ignore on the right — feels more like a "is this done?" affordance separate from the "view / hide" controls. Tom signed off.

**Follow-up filed (item 34).** Verification surfaced a real UX gap shared with the existing ignore-flow: cross-device propagation only happens at next cron tick, not on page refresh. Filed as #34 (cross-device state sync on page refresh) ahead of #33 in priority — Tom's call. Auth and reconciliation questions resolved in the same session; design note pending.
