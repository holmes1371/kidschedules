# 25. Catch self-notes / direct kid-name emails (e.g. "Everly volleyball") — 0f4a1d2 (design + flip) / ad8f1e1 (kid_names query + 9 tests) / bef3db1 (sports keyword extension + 1 pin) / 656310a (STEP-2 dynamic count) / 7d6549b (session summary)

Filed 2026-04-24 after Ellen's self-note from `ellen.n.holmes@gmail.com` to herself — Subject: `Everly volleyball`, body: `8-9am May 4-8. Sent from my iPhone` — landed inside the 60-day lookback window but matched **none** of the five existing query templates: no school-activity vocabulary, no appointment vocabulary, the sports template was missing every common school sport (`volleyball / soccer / basketball / baseball / softball / lacrosse / tennis / track / football / hockey / wrestling`), no due/deadline language, and the sender wasn't `school/district/pta/ptsa` nor was the subject a `calendar/newsletter/reminder/upcoming/schedule` form. Three converging gaps: incomplete sports list, no handling for terse self-notes, and the highest-precision signal we have — the kids' first names — was unused.

**Fix: 6th query template `kid_names`, roster-driven.** `build_queries.py` loads the roster via `roster_match.load_roster` (reuse, not duplicate the name list), builds the OR-joined parenthetical body, and inserts it into `queries` / `loose_queries` alongside the existing five with the same `after/before/exclusion` framing. Names with embedded whitespace get double-quoted defensively for forward-compat; empty roster suppresses the query (Gmail rejects `()`); single-kid roster yields `(Name)`. CLI: `--roster PATH` (default repo root) and `--no-kid-names` for diagnostic opt-out, parity with the existing `--blocklist ''` and `--no-category-filter` shapes. Existing two-pass dedup in `step2b_read_promising` (messageId + threadId, item 21) collapses overlap with the other five templates — net-new candidates are the only ones reaching the agent.

**Item 25b: sports vocabulary extension.** Independent commit (bef3db1) adding `volleyball / soccer / basketball / baseball / softball / lacrosse / tennis / track / football / hockey / wrestling` to `SEARCH_TEMPLATES["sports_extracurriculars"]`. Hygiene only — kid_names already catches the missed-email path — but closes the underlying vocabulary gap so future sports emails match without needing a kid name in the subject. Pinned by `test_sports_extracurriculars_template_covers_common_school_sports`.

**STEP-2 log dynamicized (656310a).** `main.py::step2_search_gmail` previously printed `Searching Gmail (5 queries)`; now prints `(N queries)` derived from `len(config["queries"])` so the log stays accurate as templates land or are suppressed.
