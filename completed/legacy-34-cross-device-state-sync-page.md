# 34. Cross-device state sync on page refresh (ignore + completed) — 4428700 (design note + ROADMAP [~] flip) / e2a8cf1 (apps_script.gs drops ?secret= on the three list-shape GETs) / 93d257d (client JS fetch + reconcile + localStorage schema bump + 9 tests) / e1151c9 (close-out + SHAs)

Filed 2026-04-27 from Tom — caught during #32 live verification. Closed the UX gap shared with the existing ignore-flow: state flips persisted to the Sheet immediately on click but propagated to OTHER devices only on the next cron tick (Mon / Wed / Sat 6:15 ET). Concretely: Ellen ignores or completes an event on her phone → row written to sheet → Tom opens the page on his tablet between cron ticks → he still saw old state for hours. Fix: client-side fetch on every page load, reconcile each card against fetched lists before localStorage layers on. Architecture invariant preserved — Sheet is still the single source of truth; this just shortened cross-device latency from "next cron tick" to "next refresh." Full design at `design/refresh-state-sync.md`.

**Resolved decisions (2026-04-27).** Auth Option B — drop `?secret=` on the three list-shape GETs (`?kind=ignored | completed | ignored_senders`); POSTs unchanged. Each fetched row carries only `(event_id, name, date)` — same metadata already public on the rendered page, so dropping the secret on read costs nothing in confidentiality and unblocks the client fetch (which can't carry a secret without leaking it in page JS). Reconciliation: sheet wins on refresh, with a 10-second timestamp-based grace window for in-flight POSTs. Offline write queue explicitly out of scope (tracked at #35).

**Architecture (3-tier client state).** SSR paints from cron-time JSON (status quo, fast first paint) → localStorage hydration applies overlays (status quo, this-device flips) → fetch overlays cross-device current state (NEW). The fetch fires AFTER hydration so first paint is unblocked. Three independent fetches via Promise.allSettled-style — a failure on any single list does not block reconciliation of the other two; the failed list silently degrades to SSR + hydration state.

**localStorage schema bump.** Bare ids/domains became objects of shape `{id|domain, flipped_at_iso}`. Pre-#34 bare-string entries are tolerated by the load helpers — each becomes `{..., flipped_at_iso: ""}` so the age check classifies them as stale and the first reconcile drops them. No migration code; the staleness IS the migration. Public API surface preserved: `loadIgnored()` / `loadIgnoredSenders()` / `loadCompleted()` still return bare-id arrays for existing callers; new `loadXxxEntries()` variants expose the timestamps for the reconciliation logic.

**Reconciliation rule (per card).** id in fetched list → apply flipped state, drop matching localStorage entry. id NOT in fetched AND local entry's age (now − flipped_at_iso) < REFRESH_GRACE_MS → keep the local flip (POST may be in flight or sheet GET propagation hasn't caught up). id NOT in fetched AND age ≥ grace OR no local entry → apply not-flipped (SSR may have rendered as flipped from a since-removed sheet row; we restore). The grace constant is 10s — covers in-flight POST plus typical Google sheet→GET lag. Sender reconciliation parallels event reconciliation but cards already individually-ignored (`reason="event"`) are untouched; only sender-reason cards get reconciled.

**Failure posture.** Fetch errors → silent degrade with `console.warn` breadcrumb only, no toast. Same posture as the existing webhook POST silent-degrade for dev/preview runs. Ellen rarely opens the page in low-signal places, and a "sync failed" toast on every flaky moment would be more annoying than useful. Empty `WEBHOOK_URL` (dev-preview render) early-returns before any fetch.

**Apps Script change.** Single-function patch in `doGet` adds a `PUBLIC_KINDS = ['ignored', 'ignored_senders', 'completed']` allowlist; kinds not in the list still require `?secret=$READ_SECRET`. Existing CI cron callers (the two workflow sync steps) keep working — the secret param is silently ignored along the public path. Tom redeployed Apps Script mid-feature after commit 2 lands and smoke-tested an unauthed `curl ${URL}?kind=completed`.

**Tests.** 9 new JS-substring asserts in `tests/test_process_events.py` pinning: three-kind fetch shape; absence of `secret=` in the fetch URL block; new save-helper schema with `flipped_at_iso` preservation; load-helper tolerance of pre-#34 bare entries; the 10-second grace constant; the "fetched-list wins" branch; the grace-window keep branch; silent `console.warn` fetch-failure path; dev/preview short-circuit. Total suite went from 745 to 754 passing. Existing #32 tests (which pin `loadCompleted/saveCompleted` call shapes) all kept working unchanged because the public API surface was preserved.

**CI fix-up moment, captured in pre-push protocol.** No CI failures on #34's pushes — the post-#32 protocol (full `pytest tests/ -q` on the strftime-patched copy before any push, not just `pytest -k` filtered) caught everything locally. The protocol was formalized as a memory note this session: never settle for name-filtered or single-file pytest runs when changes touch render code.

**Live verification (2026-04-27).** Tom tested the verification scenarios end-to-end and confirmed: phone-flip → tablet-refresh propagates within ~1s of fetch return for ignore-event, ignore-sender, and completed; same-device immediate refresh keeps the local flip via the grace window; cross-device unflip correctly restores the not-flipped state on refresh.
