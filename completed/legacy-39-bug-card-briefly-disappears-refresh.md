# 39. Bug: card briefly disappears on refresh due to hydration vs reconcile asymmetry — 3c4da6a

Filed 2026-05-01 from Tom — caught on the Field Trip card on the live page: refresh the page, card disappears for ~1–2s, reappears.

**Root cause.** Asymmetry between **hydration** (synchronous, applies localStorage on page load) and the post-fetch **reconcile** pass (async, runs after the Apps Script GET returns). Reconcile is timestamp-aware — only honors local entries within `REFRESH_GRACE_MS` (10s). Hydration was *not* — it applied `setIgnored(card)` for any id present in localStorage regardless of the entry's `flipped_at_iso` age. `setIgnored` does `card.style.display = "none"`, which is the only state flag that visually hides a card. So a stale localStorage entry (e.g. ignored long ago, then unignored from another device → sheet row gone but local entry persists) produced: SSR paints visible → hydration fires `setIgnored` → card disappears → ~1–2s later fetch returns, reconcile sees the id isn't in fetched and the local entry's age is ≥ grace → `setActive(card)` → card reappears. The reconcile-persist step then drops the stale entry on the same refresh, so subsequent refreshes are clean — but the one-shot flicker on each affected card was jarring.

The same asymmetry existed for sender-ignore and completed hydration. Neither uses `display:none` (sender-ignore-via-card path does, but completed hydration just adds a strikethrough class), so their flicker was less visible. Fix is symmetric in all three because the bug pattern is the same.

**Fix.** Hydration now reads the *Entries* form (`loadIgnoredEntries`, `loadIgnoredSendersEntries`, `loadCompletedEntries`) so `flipped_at_iso` is in scope, and gates `setIgnored` / `setCompleted` on `if (!(age < REFRESH_GRACE_MS)) return;`. Stale entries skip hydration and let the reconcile pass decide them — which still drops them on the same refresh, so no behavior loss for just-flipped cards (those sit within the grace window). NaN-safe via the negation form: empty / malformed `flipped_at_iso` reads as NaN, the `!(NaN < N)` test is true, entry treated as stale (correct — pre-#34 bare entries had no timestamp). A single `hydrationNow = Date.now()` is captured once and reused across all three forEach loops so a slow loop can't race the boundary across cards.

**Tests.** 1 existing test (`test_render_html_js_complete_hydrates_from_localstorage`) updated to pin the new `loadCompletedEntries` call shape (replacing the old bare-id `loadCompleted` form). 5 new pins under "ROADMAP #39: hydration is timestamp-aware to avoid flicker": entries-form switch on event-ignore (`var localIdEntries = loadIgnoredEntries();`), age-gate present in all three branches (`if (!(iAge < REFRESH_GRACE_MS)) return;`, `if (!(sAge < REFRESH_GRACE_MS)) return;`, `if (!(cAge < REFRESH_GRACE_MS)) return;`), single `hydrationNow` captured once. The negation form is pinned rather than the positive form because the negation is what makes the test fail loudly if a future edit drops the gate.

**Live verification (2026-05-01).** Tom refreshed the live page after the workflow_dispatch rebuild and confirmed the disappear/reappear flicker on the Field Trip card is gone. Signed off.
